## Commands

```bash

```

No tests, no linter configured.

## Stack

Vite + React 18, Tailwind CSS 3 (JIT), Framer Motion 11, Three.js (npm, not CDN).
Single-page portfolio — no router, no backend.

## Architecture

**Section flow.** `App.jsx` composes `<Navbar />` + `<Hero />` + `<About />` + `<Projects />` + `<Contacts />` + `<Footer />` as a single scrollable page with anchor links (`#about`, `#projects`, `#contacts`). Each section is self-contained and lives in `src/components/`.

**Projects are data-driven.** All six project entries live in `src/data/projects.js`. Each entry has `type` (used both as the visible badge label AND as the category-filter key in `Projects.jsx` → `CATEGORIES`). Supported types today: `WEB APP`, `LANDING`, `TELEGRAM BOT` — the `web` filter matches the first two. Adding a new type requires updating the matcher in `Projects.jsx`.

**Preview resolution order in `ProjectPreview.jsx`:** `video` → `image` → inline JSX fallback by `preview.kind`. `imageZoom` on a project applies `transform: scale()` to the img (used for Tezis); still composes with `object-cover`. The three inline fallback kinds (`slides`, `printer`, `telegram`) are kept as last-resort renderers if both `video` and `image` are absent.

**Projects slider logic.** Lives in `Projects.jsx`. Key state: `view` (`slider`/`grid`), `catId` (filter), `[index, dir]` (slider position + direction for enter/exit variants), `autoPlay`. Autoplay = `setInterval` at `AUTO_MS`. Any user nav (arrow/dot/drag) → `setAutoPlay(false)` + `setTimeout(PAUSE_MS)` to resume. View switch clears the pause timer and resets autoplay. View transitions use `AnimatePresence mode="wait"` at the top level.

**3D cube (`RubiksCube.jsx`).** Single `THREE.BoxGeometry` with six canvas-rendered face textures generated by `makeFaceTexture(label)`. Rotation policy matters: Y auto-rotates continuously; X is clamped to `±0.9` during drag and eases back to `BASE_X = -0.28` after release so text never flips upside-down. Don't re-introduce free X auto-rotation. All Three.js resources (textures, materials, geometries, renderer) are disposed in the effect cleanup — keep it that way.

## Design system

Palette is canonical — always use the Tailwind tokens or CSS vars, not raw hex:

| Token | Hex | Role |
|---|---|---|
| `cream` / `--cream` | `#f5f0e8` | page bg, light surfaces |
| `ink` / `--ink` | `#1a1a1a` | primary text, borders |
| `ink2` / `--ink-2` | `#444` | secondary text |
| `accent` / `--accent` | `#e63946` | primary accent (red) |
| `yellow` / `--yellow` | `#f5c842` | secondary accent |

Fonts (Google Fonts, imported in `src/index.css`): `Space Grotesk` 800 for display, `Inter` 400/500 for body, `Space Mono` for accents/labels.

**Brutalist rules, enforced throughout:**
- Borders: `2-3px solid #1a1a1a`
- Shadows: hard offset, no blur — `4px 4px 0 0 #1a1a1a` (Tailwind: `shadow-brut4`), grows to `shadow-brut6`/`shadow-brut8` on hover
- Hover motion: `translate(-2px, -2px)` + shadow step up
- Sharp corners — no `border-radius` above 4px
- Reusable primitives: `.btn-brutal` and `.tag-brutal` in `src/index.css`

When a new styled element is needed, check `tailwind.config.js` (`boxShadow.brut*`, custom colors) before inventing ad-hoc values.

## Non-obvious gotchas

- **Project cards in the slider must share height** or the slider jumps on transition. Enforced via `min-h-[540px] md:min-h-[600px]` on `<motion.article>` in `ProjectCard.jsx` plus `line-clamp-3` on the description. Don't remove either.
- **Ticker bar in `Hero.jsx`** uses 8 duplicate copies + `animate-[scroll_…]` translating `-50%`. Fewer copies leave gaps on wide viewports; more copies is fine. Mobile uses a shorter duration via responsive variant.
- **`tailwind.config.js`** has custom `boxShadow.brut{2,3,4,6,8}`, `borderWidth.3`, and the palette. New shadow sizes go there, not inline.
- **`ProjectPreview.jsx` isBot branch** paints `bg-[#17212b]` (Telegram-dark) for `TELEGRAM BOT` type. Kept even though current bot previews are SVGs that paint their own bg — lets screenshots fall back cleanly if someone swaps a raw PNG in.
- **Scroll target offset.** `section[id] { scroll-margin-top: 80px; }` in `index.css` compensates for the fixed navbar on in-page nav. Don't remove.
- **Cube is three (npm), not CDN.** The original spec called for CDN — a past decision migrated to npm. `three` is a dep, not a script tag.

## Adding a new project

1. Add entry to `PROJECTS` in `src/data/projects.js` with `id`, `num` (renumber the others if inserting mid-list), `type`, `title`, `description`, `stack`, `href`, and one of: `video` / `image` (optionally `imageZoom`) / `preview` (inline kind).
2. If using a raster/SVG preview, drop the file in `public/previews/` and reference with absolute path `/previews/filename.ext`.
3. If the project introduces a new `type` string, update `CATEGORIES` matchers in `Projects.jsx`.
